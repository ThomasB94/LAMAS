Implementation

 

We have created a program that simulates playing The Game, both for analytical purposes as well as graphical purposes. It has been made in Python with the open-source packages PyGame and mlsolver. The following image shows the graphical simulation which aims to help visualize what each player announces, and what the effect of these announcements is on the game. In this simulation, you are free to choose what the announcement type the user would like to be used (absolute, relative or none). Furthermore, the number of announcements that a player can make in a turn can be specified (1, 2 or 3 announcements).

The analytical simulation variant differs from the graphical option in that there is no visualization. This analytical variant is used to perform our experiments with, as explained in the experiments section.

Future work

 

While our simulation has proven to increase the win ratio when using public announcement logic, it is not entirely satisfactory in terms of how realistic the simulation is. The simplifications we have made provide a reasonable estimate to how public announcement logic influences The Game, but at the same time these simplifications also impose some important limitations. For example, after the cards have been dealt to both players, only 6 more cards remain in that current game. This makes our simplified version much more deterministic than the 'real' game. In other words, in our version the chance of getting the 'perfect' card for a stack is relatively high. It would be very interesting to investigate what kind of effect a decreased amount of determinism has on the parameters we have researched in this project (number of announcements and type of announcement). A possibility to research this, could be Symbolic Model Checking for Dynamic Epistemic Logic (SMCDEL)1, which improves the speed at which Kripke models can be evaluated and therefore allow for games with more cards (and thus more states) to be modelled.
